# CLAUDE.md

## 概要

このアプリケーションは`Tumiki MCP Manager`を理解するための検証用学習プロジェクトです。
シンプルな Todo アプリを題材に、以下を体験することを目的とします。

- Tumiki 経由で Claude Code と Supabase の MCP サーバーを接続し、
  AI による DB 構築・操作を実際に行う
- その過程で、MCP を一元管理し権限・操作を制御する Tumiki の価値を理解する

なお、アプリの機能的な完成度よりも、上記の体験と理解を優先する。

## 機能

- Todo の一覧表示
- Todo の追加(タイトル入力)
- Todo の完了/未完了の切り替え
- Todo の削除

過剰な機能は追加しない。最小構成を優先する。

## 技術スタック

- Next.js 15 (App Router)
- TypeScript (strict)
- Tailwind CSS

## 開発フェーズ

- **フェーズ 1**: ダミーデータで動作する Todo UI を構築する。DB 接続はまだ行わない
- **フェーズ 2**: Supabase(MCP 経由)と接続し、ダミーデータを実データに差し替える

## 基本原則

- 必要な場合、コンポーネントの末端に Client Components を使用
- データ取得は Server Components で行い、必要なデータを Client Components に props で渡す
- データ取得・更新のロジックは `src/lib/todos.ts` に集約する。
- これらの関数は **フェーズ 1 の時点から async で実装**する
  (ダミーデータを返す場合も async にし、呼び出し側は await する)。
- コンポーネントはこの関数経由でのみデータを扱い、ダミーデータを直接参照しない。
- コンポーネント、関数などはそれらが使用されるコンポーネントのできる限り近くに置き、コロケーションに従う
- 複数箇所にコンポーネントがまたがる場合は`src/components`にファイルを集約させる
- 型定義は type で統一する(interface は使わない)

## 命名規則

- ファイル名: ケバブケース(`todo-form.tsx`)
- コンポーネント名: パスカルケース(`TodoForm`)
- 関数名: キャメルケース(`createTodo`)

## データ取得の原則

- Supabase のクエリで `select("*")` は使わない。
  必要なカラムのみを明示的に指定する(例: `select("id, title, completed")`)。
  理由: 転送量の削減、意図の明示、想定外のカラム露出(機密情報の漏洩)の防止。

## やってほしくないこと

- localStorage は使わないでください
- `any`型の使用は禁止する
- DB 接続はしないでください
- 不要なライブラリは勝手に追加しないでください
- 設定ファイル(`next.config.ts`, `tsconfig.json` 等)と `.gitignore` は変更しない
- 一度に大量のコードを書かず、段階的に実装する
